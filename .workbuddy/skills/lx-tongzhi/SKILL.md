# lx-tongzhi — 触达通知生成

## 定位

`lx-tongzhi` 负责把已确认的业务事实改写成不同触达人群可直接使用的通知内容。

支持三类人群：

| 人群 | 参数 | 视角 |
|---|---|---|
| 商家 / 运营主体 | `shangjia` | 要做什么、截止时间、涉及城市/品牌、链接、配置要求 |
| 司机 | `siji` | 有什么权益、何时生效、怎么参与、注意事项 |
| 线下渠道 | `qudao` | 如何转述、如何催办、如何反馈异常、哪些口径不能说错 |

支持四类输出格式：

| 格式 | 参数 | 用途 |
|---|---|---|
| 短信 | `duanxin` | 短文本、批量导入、强约束字数 |
| Push | `push` | 标题 + 正文，突出利益点或动作 |
| 微信 / 群通知 | `weixin` | 结构化通知，可复制到微信或群 |
| 操作说明 | `caozuoshuoming` | 给执行人看的步骤清单，不作为外发文案 |

## 人群视角差异

同一份业务事实，面向不同人群时必须改变信息重心：

### shangjia

面向商家 / 运营主体，重点是“需要配合什么”。优先写清楚涉及城市、品牌、数据链接、提报/检查/反馈动作、截止时间和责任边界。不要展开司机端权益细节，也不要暴露后台字段、补贴属性或内部判断过程。

### siji

面向司机，重点是“我能获得什么、怎么参与”。优先写清楚权益、活动时间、适用城市/品牌、参与方式和查看入口。不要写商家的提报要求、后台配置步骤或内部口径。

### qudao

面向线下渠道，重点是“如何转述和推进”。优先写清楚对外口径、催办对象、截止时间、反馈方式和异常收集要求。不要给出未确认承诺，也不要照搬给司机或商家的长篇原文。

更详细的人群规则维护在 `references/audience_rules.md`。

## 输出格式差异

同一人群下，不同格式也要改变写法：

### duanxin

用于短信平台或短文本触达。只保留最关键的信息：对象、权益/事项、时间、动作、链接。避免长背景、复杂列表和多层解释。

### push

用于 App Push 或消息中心。必须包含标题和正文。标题突出利益点、截止时间或动作；正文补充生效时间、参与方式和查看入口。

### weixin

用于微信群、企微群或运营群。可以结构化分点，适合写涉及范围、执行动作、截止时间、链接和注意事项。

### caozuoshuoming

用于执行人参考。重点是步骤、检查项、异常反馈方式和完成标准。不是直接外发文案，但仍要执行对应人群禁词检查。

## 触发时先确认

除非用户已经明确给出，执行前必须确认：

1. 通知对象：`shangjia` / `siji` / `qudao`。
2. 输出格式：`duanxin` / `push` / `weixin` / `caozuoshuoming`。
3. 输入材料来源：活动 TXT、拆表汇总、日报摘要、海报元数据、用户直接粘贴的事实等。
4. 是否有明确限制：字数、标题长度、链接是否保留、是否按城市/品牌/运营主体分别生成。

不要为了补全内容编造日期、金额、奖励、城市、品牌、链接、截止时间或活动规则。

## 核心流程

1. 动态枚举并读取用户指定的真实文件；用户未给完整路径时，先用 `rg --files` / `find` / `ls` 确认文件名。
2. 抽取事实：活动名称、活动时间、城市、品牌、对象、权益、动作、截止时间、链接、注意事项。
3. 按 `audience` 切换视角，去掉该人群不该看到的内部信息。
4. 按输出格式套用对应格式和长度要求。
5. 按通知对象读取 `references/banned_terms/<audience>.md`，检查禁止词和风险词。
6. 输出通知文案和校验结果；如存在禁止词，必须先修改到不含禁止词再给最终文案。

## 禁词与风险词

禁词由人工维护在：

```text
.workbuddy/skills/lx-tongzhi/references/banned_terms/shangjia.md
.workbuddy/skills/lx-tongzhi/references/banned_terms/siji.md
.workbuddy/skills/lx-tongzhi/references/banned_terms/qudao.md
```

生成司机、商家、线下渠道外发内容前必须读取对应人群的禁词文件。禁止词不得出现在最终外发文案中；风险词出现时需要提示用户确认。

可用脚本检查：

```bash
python .workbuddy/skills/lx-tongzhi/scripts/check_terms.py \
  --audience siji \
  --text-file "待检查文案.md"
```

## 和现有 Skill 的关系

`lx-tongzhi` 是触达内容生成 Skill，不负责拆表、生成海报、发布飞书表格或发送消息。

可读取以下上游产物：

| 上游 Skill | 可读取内容 | 用途 |
|---|---|---|
| `lx-zhutichaibiao` | 拆表输出、汇总报告、各运营主体链接 | 生成商家/运营主体通知 |
| `lx-haibao` | 活动 TXT、海报产出路径、二维码校验结果 | 生成司机短信、push、配套群通知 |
| `lx-biaogetongbu` | 表格同步后的登记表/汇总表 | 生成登记完成、待处理、异常提醒 |
| `lx-dapanribao` | 日报发布计划或日报摘要 | 生成日报同步通知 |

`lx-tongzhi` 第一版只生成内容和校验报告，不直接调用其他 Skill，也不发送短信、push 或群消息。

## 输出要求

最终输出必须包含：

- 通知对象和输出格式。
- 使用的事实来源；如果来自文件，写明真实文件路径。
- 生成文案。
- 禁止词检查结果。
- 风险词检查结果。
- 未使用或缺失的关键信息。

## 常用示例

### 司机短信

```text
对象：siji
格式：duanxin
重点：权益 + 生效时间 + 参与动作
```

输出只写司机能理解和执行的信息，不出现内部补贴、后台配置、策略标签。

### 商家群通知

```text
对象：shangjia
格式：weixin
重点：涉及范围 + 截止时间 + 需要商家执行的动作 + 链接
```

适合接 `lx-zhutichaibiao` 的拆表汇总和飞书表格链接。

### 线下渠道操作说明

```text
对象：qudao
格式：caozuoshuoming
重点：转述口径 + 催办节奏 + 异常反馈路径
```

`caozuoshuoming` 可以保留执行步骤，但仍不得包含对应人群禁词清单中的词。
