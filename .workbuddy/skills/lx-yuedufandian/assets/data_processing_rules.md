# 月度返点数据处理口径

本文件记录跨月复用的数据处理口径。每月具体规则仍以当月 `规则/rules.yaml` 为准。

## 源数据入库

源 Excel 入库采用“关键字段结构化 + raw_payload 全字段留底”。

- 账单对账：写入 `lxfandian.bill_raw`，保留每行全字段 `raw_payload`，并结构化常用字段；再派生 `lxfandian.bill_agg` 做计算加速。
- 过程考核：写入 `lxfandian.proc_raw`，保留每行全字段 `raw_payload`。
- 目标拆解：写入 `lxfandian.targets`，保留每行全字段 `target_payload`。
- 开城奖励：写入 `lxfandian.open_city`，保留每行全字段 `raw_payload`。

这样后续规则新增字段时，优先从数据库重算，不再回到 Excel 重新读取。

## 开城周期剔除返佣基数

返佣基数需要剔除开城城市在开城奖励周期内的非共补 GMV。

```text
剔除前返佣基数 = 过滤强制共补和灵犀PBD后的乘客应收金额 - 春节服务费
开城周期剔除GMV = 开城城市在开城奖励日期周期内的剔除前返佣基数
最终返佣基数 = 剔除前返佣基数 - 开城周期剔除GMV
```

日期周期来自开城奖励源文件的周期字段，默认 `备注1`。账单日期来自 `分区日期(天)`。日期区间按闭区间处理，开始日和结束日都剔除。

同一品牌城市多条开城奖励记录先合并日期区间，再剔除 GMV，避免重叠日期重复扣减。

如果开城奖励记录缺少周期或周期无法解析，按当月 `rules.yaml` 的 `open_city_reward.invalid_period_policy` 处理：

- `error`：阻断计算，要求先补齐源数据。
- `warn`：该行不参与返佣基数剔除，但写入输出校验明细。
- `ignore`：该行不参与返佣基数剔除，也不写校验明细。

## TSH 增速

TSH 增速按日均值对比，不能直接用月总 TSH 对比。

```text
本月日均 TSH = 本月 TSH 总和 / 本月自然日天数
上月日均 TSH = 上月 TSH 总和 / 上月自然日天数
TSH 增速 = (本月日均 TSH - 上月日均 TSH) / 上月日均 TSH
```

例如 5 月对比 4 月时，分母分别使用 31 天和 30 天。

## 品牌曾用名

过程考核指标匹配品牌时支持曾用名。原始入库数据不改写，计算时按 `rules.yaml` 的 `brand_aliases` 把曾用名归入现名。

当前已沉淀的品牌曾用名：

| 现名 | 曾用名 |
|---|---|
| 拼哒出行 | 拼哒约车 |
| 蔚蓝滴仕 | 快来用车 |
| 蓉橙出行 | 秦汉出行 |
| 逸乘出行 | 逸乘特惠 |
| 旗妙出行极速版 | 旗妙出行 |
| 安安用车 | 安心出行 |

适用范围：

- 供需策略参与率、商家独补卖卡策略参与率等红线指标。
- TSH 增速。
- 先锋司机上线率。
- 司机客服接起率。
- 后续新增的过程考核指标，只要通过品牌字段匹配，也默认使用同一套曾用名规则。

注意：如果同一个指标表里同时出现现名和曾用名，脚本会把两者都纳入求和；当月规则复盘时需要确认这不是重复数据。

## 过程返佣 2000 单门槛

过程返佣的 2000 单门槛按运营主体出行口径日均完单判断，不按单个运营主体品牌的日均完单判断。

```text
如果运营主体出行口径日均完单 >= 2000：
  该运营主体下参与过程考核的品牌可以按过程指标结果拿过程返佣
  即使该品牌自身日均完单 < 2000

如果运营主体出行口径日均完单 < 2000：
  该运营主体下所有品牌都没有过程考核返佣
```

分开考核品牌仍然按自己的品牌城市范围计算过程指标；这里只改变“是否满足 2000 单门槛”的判断口径。

2026 年 5 月例子：拼哒出行运营主体出行口径日均完单超过 2000，因此拼哒出行运营的逸乘出行即使自身日均完单低于 2000，也可以按过程指标结果拿过程返佣。

## 规模返佣日均完单口径

规模返佣按运营主体出行口径日均完单判断阶梯，不按 GMV 判断：

```text
运营主体出行口径日均完单 = 运营主体出行品牌月总完单 / 当月自然日天数
```

同一运营主体下所有输出品牌适用同一个规模返点。

## 优行独立口径

`xx优行` 完全独立考核，不套用出行的规模、过程和完单规则。

优行完单不计入对应运营主体的出行规模阶梯、过程 2000 单门槛和完单 1000 单门槛。

出行品牌的 1000 单完单门槛按运营主体出行口径日均完单判断；运营主体达门槛后，品牌自身仍按目标表的一/二/三档日均目标判断命中档位。

## 聚合考核目标口径

同一运营主体下多个品牌都是聚合考核时，先把这些聚合品牌的一/二/三档日均目标按运营主体求和，再用运营主体聚合日均完单统一判断命中档位。

聚合组内输出品牌复用同一套规模返点、过程返点、完单返点、额外返点和最终返点。

## 先锋司机分母为 0

先锋司机上线率的分母为 0 时，按 100% 通过处理，并在过程考核 sheet 原因列标记。

## 过程指标完成度

总考核 sheet 的 `过程指标完成度` 按普通过程指标基础返点完成比例计算：

```text
过程指标完成度 = 已完成的普通过程指标基础返点 / 普通过程指标基础返点总额
```

先锋司机 LX 额外返点不计入完成度，过程返佣系数也不影响完成度。
