# FOG 管理员维护说明

## 一、维护边界

源工作区可能同时存在模板代码和管理员本地数据，因此不要把整个源目录直接发给同事。

长期推荐维护 GitHub 仓库；同事通过 `git pull --ff-only` 更新共享代码。压缩包只作为首发或临时交付方式。

## 二、发布新版

1. 修改共享 Skill、工具脚本、示例配置或文档。
2. 如新增模板文件，更新 `config/template_manifest.yaml`。
3. 检查 `config/fog_config.yaml.example` 是否包含新增配置项。
4. 运行本地检查：

```bash
python tools/fog.py migrate-config --dry-run
python tools/fog.py check
```

5. 发布到 GitHub，打 tag / Release。
6. 通知同事运行：

```powershell
git status --short
git pull --ff-only
python tools/fog.py migrate-config
python tools/fog.py init
python tools/fog.py check
```

如果 `git status --short` 有输出，先让同事发回状态，不要直接 pull。

## 三、压缩包发布

如需继续发 zip，必须通过白名单导出：

```bash
python tools/export_template.py --output "/path/to/p-fog-template" --clean
```

预览导出内容：

```bash
python tools/export_template.py --output "/path/to/p-fog-template" --dry-run
```

导出目录必须在源工作区外部。

建议检查命令：

```bash
find "/path/to/p-fog-template" -type f \( -name config.yaml -o -name personal_config.yaml -o -name database.yaml -o -name '*.xlsx' -o -name '*.xlsm' -o -name '*.docx' -o -name '*cache*.json' -o -name '*.zip' -o -name '*.pyc' -o -name '.DS_Store' \) -print
find "/path/to/p-fog-template/workspace" -type f ! -name .gitkeep -print
find "/path/to/p-fog-template/.workbuddy/skills" -maxdepth 1 -mindepth 1 -type d -exec basename {} \; | sort
```

前两个命令应该没有输出。第三个命令当前只应该看到：


## 四、白名单定义

白名单文件：

```text
config/template_manifest.yaml
```

字段含义：

- `managed_dirs`：允许导出的模板目录。
- `managed_files`：允许导出的模板文件。
- `protected_paths`：真实配置、业务数据、缓存和私有 Skill，永不导出。
- `ignored_patterns`：运行产物和临时文件，导出时跳过。

新增共享 Skill 时：

1. 把 Skill 目录加入 `managed_dirs`。
2. 只提交代码、说明、固定素材和示例结构；不要提交真实 token、API Key、缓存或运行输出。
3. 如果共享 Skill 有配置项，补 `config/fog_config.yaml.example`，代码直接读取 `config/fog_config.yaml`。
4. 把真实配置、缓存、token 路径加入 `protected_paths` 或 `ignored_patterns`。
5. 更新同事说明和管理员说明。

不分享的写库 Skill 不加入 `managed_dirs`，其个人配置放在本地 `config/personal_config.yaml`，不发布给同事。

## 五、安全规则

- 不发布 `config/fog_config.yaml`。
- 不发布 `config/personal_config.yaml`。
- 不发布 `config/database.yaml`。
- 不发布 `config/码表/` 下的 Excel。
- 不发布 `.workbuddy/skills/*/assets/config.yaml`。
- 不发布 `.workbuddy/skills/lx-haibao/output/`。
- 不发布任何 API Key、access_token、refresh_token 或密码。
- 不发布 cache、token、日志、zip、pyc、docx。
- `workspace/` 只允许发布 `.gitkeep`。
